CHAPTER 00
ROS 2とは何か？
🧩 ロボット開発の「共通ルール集」

ROS 2（Robot Operating System 2）は、ロボットを動かすプログラムを作るときに使うフレームワーク（共通の仕組み）です。カメラ・モーター・センサーなど異なる部品を1つのシステムとして連携させます。

💡 たとえ話：スマホのOS

スマホには「地図アプリ」「カメラアプリ」など、いろんなアプリが入っています。ROS 2はロボット版のOS。センサーや制御プログラムが「アプリ（ノード）」として独立して動き、互いにデータをやり取りします。

CHAPTER 01
パッケージ
📦 プログラムをまとめた「箱」

パッケージとは、関連するプログラムファイルをひとまとめにしたフォルダのことです。機能ごとに分けて管理します。

💡 たとえ話：お弁当箱の仕切り

お弁当箱には「ご飯エリア」「おかずエリア」「デザートエリア」のように仕切りがあります。パッケージも同じように、機能ごとにプログラムをきれいに整理する「仕切り」です。

🗂️ パッケージの中身（典型的な構成）
フォルダ構成
my_hello_pkg/               # パッケージ名（箱の名前）
├── my_hello_pkg/           # Pythonコードを置くフォルダ
│   ├── __init__.py         # Pythonパッケージとして認識させる空ファイル
│   ├── publisher_node.py   # 送信するプログラム
│   └── subscriber_node.py  # 受信するプログラム
├── launch/                 # 起動スクリプトを置くフォルダ
│   └── hello.launch.py
├── package.xml             # パッケージの説明書（名前・依存関係など）
└── setup.py                # インストール設定
✅ 新しいパッケージを作るコマンド
bash
$ cd ~/ros2_ws/src
$ ros2 pkg create --build-type ament_python my_hello_pkg
going to create a new package
# フォルダとファイルが自動生成される！

CHAPTER 02
ビルド と source
🔨 ビルドとは？

ビルド（build）とは、書いたプログラムをROS2が使える形に変換・整理する作業のことです。コードを書いたら必ずビルドが必要です。

💡 たとえ話：料理の「下ごしらえ」

野菜を切って、肉に下味をつけて、調理の準備をするのが下ごしらえです。ビルドも同じく、書いたコードを「ROS2が実行できる状態」に整える下ごしらえです。

⚙️ source とは？

source（ソース）とは、ビルドで作られた情報をターミナルに「読み込む（有効化する）」作業です。ビルドしただけではターミナルはパッケージを認識していません。sourceして初めて使えます。

💡 たとえ話：アプリのインストールと起動

スマホにアプリをダウンロード（ビルド）しただけでは使えません。アプリを開いて有効化（source）して初めて使えます。
source ~/.bashrc は「新しくインストールしたアプリを認識させる」操作です。

🔄 ビルド〜実行までの全体の流れ
1

ワークスペースに移動

ROS2のプロジェクトフォルダ（ワークスペース）に移動します。

bash
$ cd ~/ros2_ws
2

ビルド実行（colcon build）

colcon はROS2のビルドツールです。src/フォルダ内のパッケージをすべてビルドします。

bash
$ colcon build
Starting >>> my_hello_pkg
Finished <<< my_hello_pkg [1.23s]
Summary: 1 package finished [2.45s]
💡 特定パッケージだけビルドしたいとき（速い！）

colcon build --packages-select my_hello_pkg

3

source でパッケージを有効化

ビルドした結果をターミナルに読み込みます。ビルドのたびに必要！

bash
$ source install/setup.bash
# これでターミナルがパッケージを認識！
4

.bashrc に書いて自動化（オプション）

毎回 source するのが面倒なら ~/.bashrc に書いておくと、ターミナルを開くたびに自動実行されます。

bash
$ echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
$ source ~/.bashrc
# 次からターミナルを開くだけでOK！
⚠️ よくあるミス

コードを編集したのに動作が変わらない → source し忘れが原因のことが多いです！
編集 → ビルド → source → 実行、の順番を守りましょう。

📁 ビルド後に生成されるフォルダ
フォルダ構成
ros2_ws/
├── src/        ← 自分でコードを書く場所
│   └── my_hello_pkg/
├── build/      ← ビルド中間ファイル（触らなくてOK）
├── install/    ← ビルド成果物。source install/setup.bash で読み込む
└── log/        ← ビルドのログ（エラーが出たら確認）

CHAPTER 03
ノード
🔵「1つの仕事」をするプログラムの単位

ノードとは、1つの役割を持って動き続けるプログラムのことです。「カメラから画像を取得する」「障害物を検知する」のように、それぞれが専門の仕事を持ちます。

💡 たとえ話：会社の部署

会社には「営業部」「開発部」「総務部」のように役割が分かれた部署があります。ノードも同じく「カメラ係」「判断係」「モーター係」のように専門の仕事をしながら協力します。

📝 最もシンプルなノード（コピペで動く！）
my_hello_pkg/my_hello_pkg/hello_node.py
import rclpy                        # ROS2のPythonライブラリ
from rclpy.node import Node         # ノードの基本クラス

class HelloNode(Node):              # Nodeを継承して自分のノードを作る
    def __init__(self):
        super().__init__('hello_node')    # ノード名を設定
        self.create_timer(1.0, self.greet) # 1秒ごとに greet() を呼ぶ
        self.get_logger().info('ノード起動！')

    def greet(self):
        self.get_logger().info('こんにちは！ROS2が動いています')

def main():
    rclpy.init()            # ROS2の初期化（必ず最初に呼ぶ）
    node = HelloNode()     # ノードのインスタンスを作成
    rclpy.spin(node)        # ノードを動かし続ける（Ctrl+Cで停止）
    rclpy.shutdown()        # 終了処理

if __name__ == '__main__':
    main()

CHAPTER 04
トピック
📢 ノード間の「情報の流れ道」

トピックとは、ノードが情報をやり取りするための名前付きチャンネルです。あるノードがトピックにデータを送り（パブリッシュ）、別のノードがそれを受け取ります（サブスクライブ）。

💡 たとえ話：社内の掲示板

会社の廊下に「安全情報掲示板」「営業情報掲示板」のように名前の付いた掲示板があります。誰でも貼り出せて（パブリッシュ）、誰でも見られる（サブスクライブ）。送る側と受ける側が直接つながっていなくてもいいのが特徴！

CHAPTER 05
パブリッシュ ／ サブスクライブ
📤 パブリッシュ（Publish）

トピックにデータを送り出すこと。一定周期でメッセージを流し続けるイメージ。

📥 サブスクライブ（Subscribe）

トピックからデータを受け取ること。届いたときに自動でコールバック関数が呼ばれる。

💡 たとえ話：YouTubeチャンネル

YouTuberが動画を投稿 ＝ パブリッシュ　／　視聴者がチャンネル登録 ＝ サブスクライブ
チャンネル名 ＝ トピック名。YouTuberは視聴者が誰かを知らなくてもいい。これが「疎結合」の強みです！

📝 パブリッシャーのコード（コピペで動く！）
my_hello_pkg/my_hello_pkg/publisher_node.py
import rclpy
from rclpy.node import Node
from std_msgs.msg import String   # 文字列メッセージ型

class PublisherNode(Node):
    def __init__(self):
        super().__init__('publisher_node')
        # トピック名 '/chatter' に String型で送るパブリッシャーを作成
        # 10 = キューサイズ（一時的に溜めておけるメッセージ数）
        self.pub = self.create_publisher(String, '/chatter', 10)
        self.count = 0
        self.create_timer(1.0, self.publish_msg)  # 1秒ごとに送信

    def publish_msg(self):
        msg = String()
        msg.data = f'Hello ROS2! count={self.count}'
        self.pub.publish(msg)              # 送信！
        self.get_logger().info(f'送信: {msg.data}')
        self.count += 1

def main():
    rclpy.init()
    node = PublisherNode()
    rclpy.spin(node)
    rclpy.shutdown()
📝 サブスクライバーのコード（コピペで動く！）
my_hello_pkg/my_hello_pkg/subscriber_node.py
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SubscriberNode(Node):
    def __init__(self):
        super().__init__('subscriber_node')
        # '/chatter' トピックを購読。受け取ったら receive_msg() を呼ぶ
        self.create_subscription(
            String,           # メッセージ型
            '/chatter',        # トピック名（パブリッシャーと同じ名前！）
            self.receive_msg,  # コールバック関数
            10                # キューサイズ
        )

    def receive_msg(self, msg):   # メッセージが届くと自動で呼ばれる
        self.get_logger().info(f'受信: {msg.data}')

def main():
    rclpy.init()
    node = SubscriberNode()
    rclpy.spin(node)
    rclpy.shutdown()
⚠️ ポイント：トピック名は完全一致が必要

パブリッシャーの '/chatter' とサブスクライバーの '/chatter' が完全に同じ名前でないとデータは届きません。スペルミスに注意！

CHAPTER 06
launch ファイル
🚀 複数ノードを一発で起動する「スクリプト」

launchファイルとは、複数のノードをまとめて起動するための設定ファイルです。コマンド1行で全ノードが一斉に起動します。

💡 たとえ話：お店のオープン準備ボタン

飲食店を開店するとき「照明ON・BGM ON・レジ起動・看板ON」を毎回手動でやるのは大変です。「開店ボタン1つで全部やる」スクリプトを作るのがlaunchファイルです。

📝 launchファイルのコード（コピペで動く！）
my_hello_pkg/launch/hello.launch.py
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([

        # パブリッシャーノードを起動
        Node(
            package='my_hello_pkg',       # パッケージ名
            executable='publisher_node',   # setup.py に登録した名前
            name='my_publisher',           # 実行時のノード名
            output='screen',              # ログをターミナルに表示
        ),

        # サブスクライバーノードを起動
        Node(
            package='my_hello_pkg',
            executable='subscriber_node',
            name='my_subscriber',
            output='screen',
        ),

    ])  # これだけで2つのノードが同時に起動！
✅ launchの実行コマンド
bash
$ ros2 launch my_hello_pkg hello.launch.py
[INFO] [launch]: Default logging verbosity is set to INFO
[my_publisher]: 送信: Hello ROS2! count=0
[my_subscriber]: 受信: Hello ROS2! count=0
[my_publisher]: 送信: Hello ROS2! count=1
[my_subscriber]: 受信: Hello ROS2! count=1

CHAPTER 07
ハンズオン：ゼロから動かす全手順
🎯 目標：Pub/Sub で文字列をやり取りするシステムを作る

パッケージ作成 → コード記述 → setup.py設定 → ビルド → source → launch、という全工程を体験します。

1

ワークスペースを作る（初回のみ）

bash
$ mkdir -p ~/ros2_ws/src
$ cd ~/ros2_ws
2

パッケージを作る

bash
$ cd ~/ros2_ws/src
$ ros2 pkg create --build-type ament_python my_hello_pkg
going to create a new package
3

Pythonファイルを作る

以下のファイルを作成し、Chapter 05のコードをコピペします。

  • ~/ros2_ws/src/my_hello_pkg/my_hello_pkg/publisher_node.py
  • ~/ros2_ws/src/my_hello_pkg/my_hello_pkg/subscriber_node.py
4

setup.py にノードを登録する

setup.py の entry_points に実行可能コマンドとして登録します。

setup.py（entry_points部分）
entry_points={
    'console_scripts': [
        # '実行コマンド名 = パッケージ名.ファイル名:main関数名'
        'publisher_node  = my_hello_pkg.publisher_node:main',
        'subscriber_node = my_hello_pkg.subscriber_node:main',
    ],
},
5

launchフォルダとlaunchファイルを作る

bash
$ mkdir ~/ros2_ws/src/my_hello_pkg/launch

フォルダを作ったら hello.launch.py を作成し、Chapter 06のコードをコピペ。

さらに setup.py の先頭と data_files を追記します：

setup.py（先頭とdata_files部分）
from glob import glob  # ← 先頭に追加！
...
data_files=[
    ('share/ament_index/resource_index/packages',
        ['resource/' + package_name]),
    ('share/' + package_name, ['package.xml']),
    # ↓ launchフォルダを追加
    ('share/' + package_name + '/launch',
        glob('launch/*.launch.py')),
],
6

ビルドする

bash
$ cd ~/ros2_ws
$ colcon build --packages-select my_hello_pkg
Finished <<< my_hello_pkg [1.5s]
Summary: 1 package finished
7

source して有効化

bash
$ source install/setup.bash
8

🎉 launchで起動！

bash
$ ros2 launch my_hello_pkg hello.launch.py
[my_publisher]: 送信: Hello ROS2! count=0
[my_subscriber]: 受信: Hello ROS2! count=0
